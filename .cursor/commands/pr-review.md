# PR Review

Perform a code review
* Focus on being DRY, consistency, and best practices from a front-end perspective
* Look that it generally follows the Code Implementation Guidelines in the [frontend.mdc](mdc:clients/.cursor/rules/frontend.mdc) rules
* Look for spelling errors in strings.
* Check titles to make sure they are sentence-cased (as opposed to the standard title case).
* Check for potential bugs introduced.
* Look for potential accessibility issues.
* Focus mainly on the new changes and not so much the existing code of the changed files.
* Check for the use of `any` as a Typescript Type
